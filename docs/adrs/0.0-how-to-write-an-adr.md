# How to Write an ADR

In software development, an Architecture Decision Record (ADR) is a document that records a major, consequential decision affecting the project. An ADR explains 

- Why the decision was necessary. This is the "overview", "context", or "synopsis".  
- What the decision is. 
- Why the decision was chosen.  
- Alternatives. 
- Consequences.

# Example ADR Format

## Title: [Short description of the decision]

### Status
[Proposed, Accepted, Deprecated, Rejected]

### Context
[The problem or need driving the decision.]

### Decision
[The decision made and the reasoning behind it.]

### Alternatives Considered
[Other options and why they were not chosen.]

### Consequences
[The trade-offs and implications of the decision.]
